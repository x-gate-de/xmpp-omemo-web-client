Verwendung

Token im Header mitsenden: Authorization: Bearer <TOKEN>

  • GET /api/v1/chats — Liste deiner Chats (JID, Name, Raum, Anzahl, letzte Aktivitaet).
  • GET /api/v1/messages?hours=24 — alle Chats, letzte 24 Stunden.
  • GET /api/v1/messages?partner=<JID>&hours=24 — ein Chat (z. B. ein Raum).
  • GET /api/v1/messages?partner=<JID1>,<JID2>&hours=48 — ausgewaehlte Chats, 48 Stunden.
  • Alternativ Zeitraum per Unix-Zeit: ?from=<sek>&to=<sek>. Max. limit 20000 (Default 5000); bei mehr Treffern truncated: true.

Inkrementeller Feed (Polling)

GET /api/feed?since=<ts>&limit=200&include_outgoing=false&include_muc=true

  • Aufsteigend nach Zeit; Antwort enthaelt next_since als Cursor fuers naechste Polling (mit since=next_since weiterfragen).
  • Item-Felder: external_id (stabile Nachrichten-ID, fuer Dedup), title, body, sender, ts_source, url (Deep-Link).
  • include_outgoing=false liefert nur eingehende Nachrichten; include_muc=false blendet Raeume aus. Anhaenge erscheinen nur als Hinweis im body.

Beispiel: curl -H "Authorization: Bearer <TOKEN>" "https://chat.example.com/api/v1/messages?hours=24"

Hinweis: Die API ist genauso erreichbar wie die Web-UI; eine ggf. konfigurierte Netz-/Geo-Begrenzung der Web-UI gilt unveraendert auch fuer die API.
